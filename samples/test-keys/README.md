# Test signing material

ES256K (secp256k1) test cert chain + private key, used by `muxl-sign`'s
integration tests. Generated by `scripts/generate-test-keys.sh` —
self-signed root CA + CA-signed leaf, valid 100 years. Re-run that
script to regenerate.

**Do not use these for anything real.** The private key is committed to
the repo.

| File | Contents |
| --- | --- |
| `es256k-cert.pem` | PEM cert chain — leaf first, then root CA |
| `es256k-key.pem`  | PEM PKCS#8 leaf private key |

The c2pa-rs cert profile check (`check_certificate_profile`) requires:
- a v3 cert
- ECDSA-SHA256 signature, secp256k1 curve
- non-self-signed leaf (issuer ≠ subject)
- KeyUsage with `digitalSignature`
- ExtendedKeyUsage with one of c2pa's allowed OIDs (we use
  `emailProtection`, the same one c2pa-rs's own test fixtures use)
- AuthorityKeyIdentifier on every cert in the chain
- SubjectKeyIdentifier on any CA cert

The generation script bakes those in so the chain validates cleanly
through the c2pa-rs sign path.
